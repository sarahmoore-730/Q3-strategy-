STRATEGY — NATURAL LANGUAGE COMMANDS

Chat as the action layer

Today, getting deeper into Hatch requires users to know where to look. Natural language commands flip that — users tell us what they want in plain language, and we serve it up. No navigation, no friction, no dead ends.

FIRST FLOW IN PRODUCTION · QA AS OF JUNE 16

Caregiver access request

The first natural language command we've shipped. A parent asks to add a caregiver — Baby Agent detects the intent and opens a half sheet inline, without ever sending them to Settings.

BEFORE
Parent asks how to add a caregiver. Chat responds with step-by-step instructions. Parent leaves chat, navigates through Settings, enters an email.

WHAT WE'RE BUILDING TOWARD

The pattern works. Now we scale it.

Every time a user would have to leave chat to do something in the app, we can do it for them instead. Here's the Q3 use case list.

DEVICE SETTINGS
  • Change clock display (dim / on / off)
  • How to change the time
  • Change age to adjusted age
  • Set baby's birthday in the app
  • Update WiFi / reconnect
  • Rename the device
SLEEP LOGGING
  • Turn off auto sleep detection
  • Where can I see the sleep logs?
  • Turn off nap reminders
  • Show me an example sleep log
SOUND & LIGHT CONTROL
  • Turn off light, keep sound on
  • Turn on just the light, no sound
  • Adjust light level from my phone
  • Play something good to sleep (by age)
  • What sound/light combo is best for night?
  • Best color for midday nap (toddler)
SCHEDULED CONTENT
  • Set a schedule for lights by time of day
  • Can you set that up for me?
  • Turn off schedule
  • Set an alarm without sound
TIME FOR BED / TIME TO RISE
  • Set up red light / green light
  • Set the Hatch alarm
  • Set up Time to Rise light
  • She keeps getting out of bed — help me set this up
CONTENT & TUCK-INS
  • Get a sleep guide
  • Make a tuck-in
  • Add a custom tuck-in
  • Take me to tuck-ins

HOW WE PRIORITIZE USE CASES
01
Request volume
How often are users asking for this? High-volume questions get the most impact per use case shipped.
02
Language detectability
How reliably can we recognize the intent? Clear, common phrasings ship faster with higher confidence.
03
Engineering lift
What does it cost to wire up each use case? Dedicated resourcing lets us move faster against this constraint.
