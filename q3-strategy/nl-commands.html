STRATEGY — NATURAL LANGUAGE COMMANDS

Chat as the action layer

Today, getting deeper into Hatch requires users to know where to look. Natural language commands flip that — users tell us what they want in plain language, and we serve it up. No navigation, no friction, no dead ends.

TL;DR

What we're doing: Turning chat into the action layer for Hatch. When a parent asks to do something — "add a caregiver," "dim the clock," "turn off the light but keep the sound" — Baby Agent recognizes the intent and handles it right in the conversation, instead of replying with instructions and sending them off to find the setting themselves.

Why it matters: Every "how do I…" is a moment a parent has to leave chat, dig through the app, and sometimes give up. Doing it for them removes that friction, meets the rising "just do it for me" expectation, and makes chat the fastest way to use Hatch — which deepens engagement and repeat use. The caregiver flow already proved the pattern: 7 clicks down to 0, on a question 134 parents asked in 90 days.

OUR PROCESS

How we work through use cases

Two loops: first we decide which use cases are worth building, then we build the ones that make the cut.

1 · DECIDING WHAT TO BUILD
01
Semantic matching
Analyze real conversations to determine the intent a parent is expressing — even across the many different ways they phrase it.
02
Sizing use cases
Count how often each intent actually comes up — via would-have-matched reports over the last 90 days — to gauge real demand.
03
Prioritization
Weigh volume against language detectability and engineering lift to decide which use cases to build first.

THE PRIORITIZATION FACTORS
01
Request volume
How often are users asking for this? High-volume questions get the most impact per use case shipped.
02
Language detectability
How reliably can we recognize the intent? Clear, common phrasings ship faster with higher confidence.
03
Engineering lift
What does it cost to wire up each use case? Dedicated resourcing lets us move faster against this constraint.
2 · BUILDING A USE CASE
01
Map the language
Pull the real phrasings from the semantic-matching work and teach Baby Agent to detect the intent reliably.
02
Design & build the inline action
Decide what "doing it for them" looks like — a half sheet, a control, a completed change — and wire it up to the app.
03
QA, ship & monitor
Validate against the phrasing set, release to production, and have consultants watch the first live conversations.
WHAT WE'RE BUILDING TOWARD

The pattern works. Now we scale it.

Every time a user would have to leave chat to do something in the app, we can do it for them instead. Here's the Q3 use case list.

DEVICE SETTINGS
  • Change clock display (dim / on / off)
  • How to change the time
  • Change age to adjusted age
  • Set baby's birthday in the app
  • Update WiFi / reconnect
  • Rename the device
SLEEP LOGGING
  • Turn off auto sleep detection
  • Where can I see the sleep logs?
  • Turn off nap reminders
  • Show me an example sleep log
SOUND & LIGHT CONTROL
  • Turn off light, keep sound on
  • Turn on just the light, no sound
  • Adjust light level from my phone
  • Play something good to sleep (by age)
  • What sound/light combo is best for night?
  • Best color for midday nap (toddler)
SCHEDULED CONTENT
  • Set a schedule for lights by time of day
  • Can you set that up for me?
  • Turn off schedule
  • Set an alarm without sound
TIME FOR BED / TIME TO RISE
  • Set up red light / green light
  • Set the Hatch alarm
  • Set up Time to Rise light
  • She keeps getting out of bed — help me set this up
CONTENT & TUCK-INS
  • Get a sleep guide
  • Make a tuck-in
  • Add a custom tuck-in
  • Take me to tuck-ins
