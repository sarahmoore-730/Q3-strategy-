Spec the overall architecture — injection mechanism + trigger framework together
Before anything ships, we need the full architecture in place. That means speccing the injection mechanism (how a message enters chat from the backend) and the trigger framework (who gets what, when, and how we prevent over-messaging) as one unified system — not two separate things. Content cards are the first use case, but the architecture has to be built to scale to multiple triggers and user segments.
